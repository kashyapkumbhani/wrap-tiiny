üîß How It Works

1. Auto-Injection into Subdomains

When someone visits any subdomain (e.g., blog.example.com), the system automatically injects this code:

<script> window.ANALYTICS_CONFIG = { storageEndpoint: '/api/analytics/visit', analyticsEndpoint: '/api/analytics/dashboard', trackingDelay: 100, respectDoNotTrack: true, trackIndividually: true, enableDashboard: false, // No popup on user sites autoTrack: true, siteId: 'abc12345', siteName: 'blog' }; </script> <script src="/static/analytics.js"></script>

2. Data Collection Process

  1. Auto-Start: Script runs immediately when page loads
  2. IP Detection: Fetches visitor's IP info from Cloudflare Radar API
  3. Browser Data: Collects screen size, language, timezone, referrer
  4. Site Context: Identifies subdomain and visit type (direct, search, social, etc.)
  5. Storage: POSTs all data to /api/analytics/visit

3. Dashboard Integration

Your dashboard now includes Analytics buttons for each site that show:

  • üìà Total visits and unique visitors
  • üåç Geographic breakdown by country
  • üö¶ Traffic sources (direct, search, social, referral)
  • üë• Recent visitor list with locations and timestamps
  • üìä Data export functionality

üóÉÔ∏è Database Schema

Analytics data is stored in the visits table:

CREATE TABLE visits ( id TEXT PRIMARY KEY, site_id TEXT NOT NULL, -- Page Info url TEXT NOT NULL, hostname TEXT NOT NULL, subdomain TEXT, -- Visitor Info (from Cloudflare) ip_address TEXT, country TEXT, city TEXT, asn INTEGER, colo TEXT, -- Browser Info user_agent TEXT, language TEXT, timezone TEXT, screen_resolution TEXT, visit_type TEXT, created_at DATETIME DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (site_id) REFERENCES sites(id) );
